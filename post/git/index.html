Git

@布忧鱼 · Aug 11, 2019 · 1 min read

1、基本使用

git init把目录变成可管理的仓库
git add readme.txt把文件添加到仓库
git commit -m “wrote a readme file” 把文件提交到仓库，可以一次提交很多文件
git status掌握仓库的当前状态
git diff readme.txt查看文件修改的状态
git log查看历史记录 可选参数 –pretty=oneline

2、版本回退

git reset –hard HEAD^ 回退上一个版本，HEAD^^ 两个，HEAD~100
git reset –hard 版本号
git reflog记录每一个版本
git diff HEAD – readme.txt 工作区和最新版本的区别

3、撤销修改

git checkout – file 丢弃工作区的修改，让这个文件回到git commit或git add的状态，把工作区内容回退
如果以及git add之后使用git reset HEAD readme.txt可以把add的修改回退到工作区，把暂存区内容回退

4、删除

当工作区的文件被删除之后，工作区和版本库不一致。
（1）、从版本库删除文件
git rm test.txt
git commit -m “remove test.txt”
（2）、同步被删除的文件到工作区
git checkout – test.txt 使用版本库里的版本替换工作区中的版本

5、远程仓库

ssh-keygen -t rsa -C “youremail@example.com”创建SSH key
把本地仓库推送出去:
git remote add origin git@github.com:buyouyu/learngit.git
git push -u origin master之后就不需要再加-u参数了。
git clone git@github.com:buyouyu/gitskills.git

6、分支管理

创建dev分支并且切换到dev
git checkout -b dev 等价于git branch dev和git checkout dev切换分支
git branch 查看分支
git merge dev合并分支
git branch -d dev 删除分支
