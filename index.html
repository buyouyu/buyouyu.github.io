Vim

@布忧鱼 · Aug 12, 2019 · 1 min read

1、查找文本 /abc 从当前开始查找abc /^abc 查找以abc为首的行 /abc$ 查找以abc为尾的行 ?abc 向后查找字符串abc 查找区分大小写，忽略大小写”:set ic”,调整回来输入”:set noic” 2、替换文本 ：%s/root/tree/g把所有root替换成tree 3、删除文本 x 删除光标所在字符 dd 删除光标所在行 ndd 删除包括此行的后n行文本 dG 删除包括此行一直到文件末尾的内容 D 删除光标位置到行尾的内容 注意删除后的文本保留在剪切板中 4、复制文本 p 将剪切板中的内容复制到光标后 P 将剪切板中的内容复制到光标前 yy 将光标所在行复制到剪切板，此命令前可加数字n，可复制多行 yw 将光标所在单词复制到剪切板 5、常用快捷键 在光标所在行按J就可以连接下一行 0或者^ 光标移动到首行 $ 光标移动至当前行的行尾 gg 光标移动到文件开头 G 光标移动到末尾 :n 编辑模式下光标快速定义到指定行的行首 6、多窗口编辑 vim -o test1.txt test2.txt “Ctrl+WW” 切换窗口 :r 被复制的文件名 将文件全部复制到光标所在处

Git

@布忧鱼 · Aug 11, 2019 · 1 min read

1、基本使用 git init把目录变成可管理的仓库 git add readme.txt把文件添加到仓库 git commit -m “wrote a readme file” 把文件提交到仓库，可以一次提交很多文件 git status掌握仓库的当前状态 git diff readme.txt查看文件修改的状态 git log查看历史记录 可选参数 –pretty=oneline 2、版本回退 git reset –hard HEAD^ 回退上一个版本，HEAD^^ 两个，HEAD~100 git reset –hard 版本号 git reflog记录每一个版本 git diff HEAD – readme.txt 工作区和最新版本的区别 3、撤销修改 git checkout – file 丢弃工作区的修改，让这个文件回到git commit或git add的状态，把工作区内容回退 如果以及git add之后使用git reset HEAD readme.txt可以把add的修改回退到工作区，把暂存区内容回退 4、删除 当工作区的文件被删除之后，工作区和版本库不一致。 ####（1）、从版本库删除文件 git rm test.txt git commit -m “remove test.txt” ####（2）、同步被删除的文件到工作区 git checkout – test.
